# FET-PET/MRI Tyrosine Kinetics Analyzer

Автоматический пайплайн анализа кинетики накопления [FET](https://en.wikipedia.org/wiki/Fluoroethyltyrosine) (фторэтилтирозин, <sup>18</sup>F-FET) по данным ПЭТ/МРТ.

Поддерживает два режима:
- **Static** — 3 статические серии (20, 40, 60 мин)
- **Dynamic** — 3 динамические 4D-серии с переменным временем экспозиции (до 38 временных точек)

---

## 1. Физика: от Bq/ml к TBR

### SUL (SUV<sub>lbm</sub>)

Сканер выдаёт значения в Bq/ml. Пайплайн конвертирует их в **SUL** — стандартизированную величину накопления (SUV, нормализованный на lean body mass):

```
SUL = A_voxel [Bq/ml] · LBM [g] / InjectedDose [Bq]
```

где **LBM** (lean body mass) вычисляется по формуле [James (1976)](https://pubmed.ncbi.nlm.nih.gov/1276566):

| Пол | Формула |
|-----|---------|
| Мужской | LBM = 1.10 · W − 128 · (W/H)² |
| Женский | LBM = 1.07 · W − 148 · (W/H)² |

*W — масса тела (кг), H — рост (см)*

SUL предпочтительнее SUV<sub>bw</sub> (на массу тела), потому что жировая ткань не накапливает FET — высокая масса тела завышает SUV, но не влияет на SUL.

### TBR (Tumor-to-Background Ratio)

**TBR** — главная количественная метрика. Для каждого временного среза вычисляется как:

```
TBR_t(x, y, z) = SUL_t(x, y, z) / SULmean_t
```

где **SULmean** — усечённое среднее (trimmed mean) всех ненулевых вокселей внутри мозговой маски:

1. Берутся все SUL > 0 внутри brain mask
2. Сортируются по возрастанию
3. Отбрасываются крайние 2.5% с каждой стороны (trim tails)
4. Вычисляется среднее оставшихся значений

Усечение хвостов защищает от: (а) занижения среднего из-за CSF/воздуха (левый хвост) и (б) завышения из-за единичных горячих пикселей (правый хвост).

**Физический смысл SULmean**: среднее фоновое накопление FET в здоровой ткани мозга. **TBR ≈ 1** в нормальной ткани; значения > 2 считаются подозрительными.


### Пороговые значения

| Параметр | По умолчанию | Смысл |
|----------|-------------|-------|
| `--sul-threshold 2.0` | Минимальный SULmax для значимой области |
| `--tbr-threshold 2.0` | Минимальный TBR (хотя бы в одной точке) |
| `--tbr-delta-threshold 0.3` | Минимальное изменение TBR за время наблюдения для классификации тренда |
| `--min-cluster-size 45` | Минимальный размер связной области (в вокселях) |

**Клинический контекст:** TBR > 2.0–2.5 обычно считается порогом для дифференцировки опухоль/здоровая ткань при FET-ПЭТ. TBR > 2.7—3.0 указывает на high-grade глиому с высокой специфичностью (Pauleit et al., 2005; Rapp et al., 2013).


## 2. Карта slope (наклон)

### Формула

Для каждого вокселя вычисляется линейная регрессия TBR(t) vs время:

```
slope = Σ[(t_i − t̄) · (TBR_i − TBR̄)] / Σ(t_i − t̄)²
```

где:
- `t_i` — временны́е точки (20, 40, 60 мин в статике; до 38 точек в динамике)
- `t̄` — среднее время трёх точек (= 40 мин в статике)
- `TBR_i` — значение TBR в вокселе в момент t_i
- `TBR̄` — среднее TBR по трём точкам для данного вокселя

В **динамическом** режиме формула та же, но суммирование идёт по всем 38 временным точкам — получается более точный и информативный slope.

### Единицы измерения

**TBR/мин** — на сколько в среднем изменяется TBR за одну минуту.

### Что означает slope

| slope | Тренд | Тип кривой |
|-------|-------|-----------|
| > +0.0075 TBR/мин | Накопление | **Rising** (восходящий) |
| < −0.0075 TBR/мин | Вымывание | **Falling** (нисходящий) |
| между ±0.0075 | Стабильный | **Plateau** (плато) |

Порог ±0.0075 TBR/мин получается из `tbr_delta_threshold / time_span = 0.3 / 40 мин`.

**Физиологический смысл:** FET вводится в t=0 (сразу после начала сканирования). Сканирование идёт непрерывно (listmode), аппарат реконструирует три статических изображения из временных окон вокруг 20, 40 и 60 мин. Каждая «статическая» точка — это усреднение сигнала за ~5–10 мин окно (для улучшения SNR), а в анализе она трактуется как мгновенное значение в центре окна. За 40 мин между первой и последней точкой «значимым» считается изменение TBR на ΔTBR ≥ 0.3 — примерно 10–15% от типичного опухолевого TBR. Это отсекает флуктуации фона (кровоток, метаболизм здоровой ткани) от реального опухолевого тренда.


## 3. Классификация кривых

Каждый воксель, прошедший порог по SULmax и TBR, классифицируется в одну из трёх категорий:

| Класс | Название | Условие | Клиническая интерпретация |
|-------|----------|--------|--------------------------|
| 1 | **Rising** | slope > +threshold | Накопление FET со временем. Типично для low-grade глиом (LGG) и реактивных изменений (воспаление, радионекроз). Отражает продолжающийся транспорт аминокислот в клетки. |
| 2 | **Falling** | slope < −threshold | Вымывание FET. Характерно для high-grade глиом (HGG, глиобластома). Связано с быстрым насыщением транспортёров LAT1 и последующим вымыванием меченой аминокислоты. |
| 3 | **Plateau** | \|slope\| ≤ threshold | Стабильный уровень. Промежуточный паттерн — может наблюдаться при глиомах Grade II–III и в пограничных зонах. |

**Важно:** классификация работает для статического режима (3 точки). В динамическом режиме (38 точек) кривые информативнее, и визуальная оценка формы кривой часто полезнее формальной трёхклассовой классификации.


## 4. Динамический анализ

В динамическом режиме пайплайн обрабатывает три 4D DICOM-серии, каждая содержит несколько временных фреймов (суммарно 38):

| Серия | Фреймы | Экспозиция | Время | Суммарно |
|-------|--------|------------|-------|----------|
| 1 | 12 + 6 + 3 + 5 + 4 | 5/10/20/60/180 с | 0–20 мин | 30 фреймов |
| 2 | 4 | 300 с | 20–40 мин | 34 фрейма |
| 3 | 4 | 300 с | 40–60 мин | 38 фреймов |

После склейки получается 4D массив **SUL(x, y, z, t)** для 38 временных точек. Далее:

1. **TBR_4D(x, y, z, t)** = SUL(x, y, z, t) / SULmean(t) — для каждого фрейма отдельно
2. **slope(x, y, z)** — линейная регрессия по всем 38 точкам
3. **TBRmax(x, y, z)** — максимум TBR по всем фреймам
4. **Классификация** — rising/falling/plateau по тому же порогу


## 5. Пост-обработка

### Connected component фильтрация

После классификации применяется фильтр связных компонент: области размером меньше `--min-cluster-size` (по умолчанию 45 вокселей) удаляются. Фильтр работает независимо для каждого класса (rising, falling, plateau) — маленькая rising-область рядом с большим plateau не объединяется с ним, а удаляется.

Это убирает шумовые воксели и оставляет только анатомически значимые кластеры.

### Brain mask

Если загружена T1 МРТ, для skull-stripping используется нейросеть **antspynet** (brain extraction). Без T1 — fallback на пороговую маску по PET. Brain mask нужна для: (а) вычисления SULmean только по мозговой ткани (исключая череп и фон), и (б) ограничения connected component анализа.


## 6. Клиническая интерпретация

Типичный протокол FET-ПЭТ с тремя статическими точками (20-40-60 мин) позволяет:

1. **Отличить опухоль от неопухолевых изменений** — TBR > 2.0—2.5 с высокой вероятностью указывает на неопластический процесс
2. **Оценить грейд опухоли** — rising-паттерн характерен для LGG, falling — для HGG
3. **Мониторить рецидив** — появление rising- или plateau-паттерна в ранее «холодной» зоне
4. **Отличить радионекроз от рецидива** — оба могут давать TBR > 2.0, но радионекроз чаще показывает rising-паттерн

Динамический протокол (38 точек) дополнительно позволяет:
- Построить полную кривую TBR(t) и оценить временной пик TBR
- Более точно классифицировать кривые (slope по 38 точкам надёжнее, чем по 3)
- Анализировать форму кривой (time-to-peak, скорость накопления на ранней фазе)

### Референсы

- Pauleit D et al. *O-(2-[18F]fluoroethyl)-L-tyrosine PET combined with MRI improves the diagnostic assessment of cerebral gliomas.* Brain, 2005.
- Rapp M et al. *Diagnostic performance of 18F-FET PET in newly diagnosed cerebral lesions suggestive of glioma.* J Nucl Med, 2013.
- Lohmann P et al. *FET PET radiomics for differentiation of pseudoprogression from early tumor progression in glioma.* Neuro-Oncology, 2018.

## 7. Запуск

### Static

```bash
# Базовый
python run_pipeline.py --t20 <20мин> --t40 <40мин> --t60 <60мин> -o <выход>

# С T1 (antspynet skull-stripping)
python run_pipeline.py --t20 <20мин> --t40 <40мин> --t60 <60мин> --t1 <T1> -o <выход>

# Пороги
python run_pipeline.py ... --tbr-threshold 1.8 --sul-threshold 1.8
```

### Dynamic

```bash
python run_pipeline.py --dynamic --dyn1 <серия1> --dyn2 <серия2> --dyn3 <серия3> -o <выход>
```

### Viewer (Qt5)

```bash
python viewer_qt.py <выходная_папка>
```
